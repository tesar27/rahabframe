**Reality check:** For biomechanics/ЛФК analysis with rich context, these small models will feel limited. Better to use **OpenAI/Anthropic API** during development and revisit self-hosting only if costs become a problem.

**4GB RAM / 2 vCores** — tight but workable, with caveats:

| Component | Fits? |
|---|---|
| FastAPI + MediaPipe | ✅ ~500MB-1GB RAM |
| ffmpeg frame extraction | ✅ CPU-heavy but manageable |
| PostgreSQL + Redis | ✅ ~300-500MB |
| Celery worker | ✅ 1 worker only |
| LLM (local) | ❌ No — use OpenAI/Anthropic API |

**Verdict: Yes for processing, but:**
- Process videos **sequentially**, not in parallel — 2 concurrent jobs will OOM
- Keep Celery to **1 worker** (`--concurrency=1`)
- Use **OpenAI/Anthropic API** for AI (no local model on 4GB)
- 120GB storage is fine for video files

The bottleneck will be **speed** — a 1-minute video at 10fps through MediaPipe on 2 vCores will take ~2-5 minutes to process. That's acceptable if you run it async (queue + notify user when done).
