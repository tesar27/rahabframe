## ЛФК AI App — Architecture & Tech Suggestions

Great project idea. Here's a practical breakdown of how to build this.

---

### Motion Capture / Pose Estimation

The core tech choice. Two main options:

**Option A — Client-side (browser)**
| Library | Pros | Cons |
|---|---|---|
| **MediaPipe Pose** (Google) | 33 landmarks, WASM, runs in browser, great accuracy | Heavier bundle |
| **TensorFlow.js + MoveNet** | Lightweight, fast (Lightning/Thunder models) | Fewer keypoints |

**Option B — Server-side (Python)**
| Library | Pros | Cons |
|---|---|---|
| **MediaPipe Python** | Same 33 landmarks, easy to use | Requires backend processing |
| **YOLOv8 Pose** (Ultralytics) | Very accurate, 17 COCO keypoints | GPU recommended |
| **OpenPose** | Most accurate | Heavy, slow, complex setup |

**Recommendation: MediaPipe on the Python backend.** Process uploaded videos server-side, extract keypoints per frame, store them as JSON. This gives you reproducible analysis without browser compatibility headaches.

---

### Movement Comparison Algorithm

After extracting keypoints (x, y, z per joint per frame), you need to compare two sequences:

1. **Normalize poses** — scale by body height, center at hips (removes camera distance differences)
2. **DTW (Dynamic Time Warping)** — compare movement sequences even if they differ in speed/duration
3. **Joint angle analysis** — calculate angles at knees, elbows, spine, hips — much more meaningful than raw coordinates
4. **Velocity/smoothness metrics** — detects tremors or jerky movements

```python
# Example: joint angle comparison
import numpy as np
```

### AI Chat Integration

Use the extracted pose data + progress scores as **context** for the LLM:

```python
context = f"""
Patient: child, age 7
Exercise: squat
Video 1 (baseline): knee angle avg 95°, symmetry score 0.62, balance score 0.71
Video 2 (follow-up): knee angle avg 112°, symmetry score 0.78, balance score 0.84
DTW similarity improvement: 34%
Weakest area: left hip flexion angle lagging by ~15°
"""

prompt = f"{context}\n\nSuggest specific ЛФК exercises to address the remaining issues."
```

**GPT-4o or Claude 3.5 Sonnet** works best — they can reason about biomechanics if given structured data.

### Processing Flow

```
1. User uploads Video A + Video B
2. FastAPI queues processing jobs (Celery)
3. For each video:
   a. ffmpeg extracts frames at ~10fps
   b. MediaPipe processes each frame → 33 landmarks
   c. Calculate joint angles per frame
   d. Store keypoint sequences in DB
4. Compare sequences:
   a. Normalize both sequences
   b. Run DTW per joint group
   c. Generate progress scores
5. Feed scores to LLM → generate report + suggestions
6. Display: side-by-side video + skeleton overlay + charts + AI chat
```

### Biggest Challenges to Be Aware Of

- **Camera angle consistency** — ideally both videos should be filmed from the same angle. You'll need to document this requirement for the user or build angle detection.
- **Child proportions** — MediaPipe is trained on adults; accuracy on small children can degrade. Consider adding a confidence threshold check.
- **Video quality** — low light / occlusion breaks pose estimation. Build graceful fallback messaging.
- **Privacy** — videos of children are sensitive data. Ensure GDPR/local compliance, encrypted storage, clear data retention policies.
